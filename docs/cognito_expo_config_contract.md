# Cognito Expo Config Contract

Defines the public configuration that the Expo app may consume from the
Cognito CDK stack. These values are identifiers, public URLs, and route paths.
They are environment-specific and should not be hardcoded in committed app source.

No Cognito app client secret is created for Expo.

## Source Stack

The contract is produced by `apps/infra`, an AWS CDK v2 TypeScript app. The
Cognito stack is defined in `apps/infra/lib/cognito-stack.ts` and configured by
`apps/infra/lib/config.ts`.

Deployment input values are loaded from the selected environment's ignored
file: `apps/infra/.env.dev` or `apps/infra/.env.prod`. The committed matching
example file lists the required variable names without real values. CDK
commands must also select `-c environment=dev|prod` explicitly.

## Public Variables

| CDK output key                  | Expo variable                       | Purpose                                      |
| ------------------------------- | ----------------------------------- | -------------------------------------------- |
| `ExpoPublicAwsRegion`           | `EXPO_PUBLIC_AWS_REGION`            | AWS region containing the Cognito user pool. |
| `ExpoPublicCognitoUserPoolId`   | `EXPO_PUBLIC_COGNITO_USER_POOL_ID`  | Cognito user pool identifier.                |
| `ExpoPublicCognitoClientId`     | `EXPO_PUBLIC_COGNITO_CLIENT_ID`     | Public Cognito app client identifier.        |
| `ExpoPublicCognitoDomain`       | `EXPO_PUBLIC_COGNITO_DOMAIN`        | Hosted Cognito domain used for OAuth.        |
| `ExpoPublicCognitoIssuer`       | `EXPO_PUBLIC_COGNITO_ISSUER`        | JWT issuer URL for this user pool.           |
| `ExpoPublicCognitoRedirectPath` | `EXPO_PUBLIC_COGNITO_REDIRECT_PATH` | OAuth callback path used by the mobile app.  |
| `ExpoPublicCognitoLogoutPath`   | `EXPO_PUBLIC_COGNITO_LOGOUT_PATH`   | OAuth logout path used by the mobile app.    |

CloudFormation output logical IDs must be alphanumeric, so the CDK output keys
use `ExpoPublic...` names. Each output description is the exact
`EXPO_PUBLIC_...` variable name it feeds.

## Local Expo Delivery

After deploying the Cognito stack, create or update `apps/tarot/.env.local` from
`apps/tarot/.env.local.example`.

Use the CDK stack outputs as the source of truth:

```sh
aws cloudformation describe-stacks \
  --stack-name <stack-name> \
  --query 'Stacks[0].Outputs'
```

Copy each output value into the matching `EXPO_PUBLIC_...` variable listed in
the table above. `apps/tarot/.env.local` is ignored by git and should remain
local-only.

The CDK outputs are the source of truth after deployment. Do not manually invent
or hardcode Cognito IDs, domains, issuers, or paths in the mobile app.

## EAS Delivery

For preview and testing builds, create the same public variables in the EAS
environment used by the build profile. Use plain text visibility because these
values are embedded in the client bundle and must be treated as public.

Until CI is added, manually sync EAS values from the deployed CDK stack outputs
whenever the Cognito stack is replaced or its domain/client identifiers change.

## Redirect Path Contract

The CDK stack outputs route paths rather than full mobile callback/logout URLs
for Expo. The mobile auth runtime is responsible for constructing platform-
specific redirect URIs from these paths and the app scheme.

Cognito callback and logout URLs configured in the selected
`apps/infra/.env.<environment>` file must match the URIs generated by the Expo
auth runtime exactly for that environment's build context.

## Security Boundary

The Expo app may read only the variables listed in this contract. Do not add a
Cognito client secret, AWS credentials, private API keys, or other privileged
values to `EXPO_PUBLIC_...` variables, `apps/tarot/.env.local`, or EAS client
environment variables.
