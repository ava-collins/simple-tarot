# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

### Monorepo-wide

```sh
yarn lint                     # ESLint across all apps and packages
yarn lint:fix                 # Auto-fix lint errors
yarn build                    # Lint then build types for all workspaces
yarn knip                     # Find unused exports/files
yarn storybook                # Start Storybook dev server (packages/ui)
yarn build-storybook          # Build static Storybook
```

### Tarot mobile app (apps/tarot)

```sh
yarn ios                      # Build and run on iOS simulator
yarn workspace tarot start    # Start Expo dev server
yarn workspace tarot test     # Run vitest tests
```

### Reading API (apps/graph-api)

```sh
yarn graph:dev                # Start REST API server with nodemon + ts-node (port 4000)
yarn graph:build              # Compile TypeScript to dist/
yarn graph:start              # Start compiled server
yarn corpus:build             # Transform assets/ignore/corpus-source.json → corpus/ + src/data/
yarn corpus:upload            # Upload corpus to S3 and trigger Bedrock KB ingestion sync
```

Requires `apps/graph-api/.env` populated with `AWS_REGION`, `BEDROCK_KB_ID`, `BEDROCK_KB_DATA_SOURCE_ID`, `CORPUS_BUCKET_NAME` (from CDK outputs) and `BEDROCK_MODEL_ID_HAIKU` / `BEDROCK_MODEL_ID_SONNET`.

### Infrastructure (apps/infra)

```sh
yarn workspace infra build    # Compile CDK app
yarn workspace infra test     # Run CDK assertion tests
yarn workspace infra cdk synth  # Synthesize CloudFormation (requires apps/infra/.env)
yarn bedrock:deploy           # Deploy BedrockInfra then BedrockKb (90s wait between)
yarn bedrock:destroy          # Destroy BedrockKb then BedrockInfra
```

### Hooks package (packages/hooks)

```sh
yarn workspace @simpletarot/hooks test   # Run vitest with coverage
```

## Architecture

This is a **Yarn 4 workspace monorepo** with two apps and two shared packages:

```
apps/
  tarot/       React Native + Expo (expo-router) mobile app
  graph-api/   Express v5 REST API (Bedrock RAG reading generation)
  infra/       AWS CDK v2 app (Cognito + Bedrock infrastructure)
packages/
  hooks/       Shared React hooks (Apollo Client for auth/account state)
  ui/          Shared React Native component library (Storybook)
```

### Data flow

`packages/hooks` → `packages/ui` → `apps/tarot`

- `packages/hooks` exports domain hooks (`useLoginForm`, `useSvgCards`, `useInstructions`, etc.) that call Apollo Client queries/mutations
- `packages/ui` imports hooks and exports fully-composed React Native screens and components (`LoginScreen`, `SignupScreen`, `CognitoSignInScreen`, etc.) via Storybook atoms/molecules/organisms/screens structure
- `apps/tarot` uses expo-router for file-based routing; screens import from `@simpletarot/ui` and auth state is managed via `AuthProvider` in `src/app/_layout.tsx`

### Reading API (graph-api)

- Express v5 REST API, no GraphQL or Neo4j
- Single endpoint: `POST /reading` — accepts `{ items: [{ cardIndex, spreadPositionIndex, reversed }] }`, returns `{ cards, synthesis }`
- Card and spread-position metadata loaded from `src/data/cards.json` and `src/data/spread-positions.json` (generated by `yarn corpus:build`)
- Per-card readings generated via AWS Bedrock Knowledge Base retrieval + Claude Haiku invocation
- Synthesis paragraph generated via Claude Sonnet after all cards are processed
- Corpus documents live in `corpus/` (generated); S3 bucket and KB provisioned by `BedrockInfraStack` + `BedrockKbStack` in `apps/infra`
- Supports SSL in production; dev runs on port 4000 without SSL

### Auth

- Cognito hosted UI + authorization-code OAuth flow via `expo-auth-session`
- `apps/infra` provisions the Cognito user pool and outputs `EXPO_PUBLIC_*` variables
- After CDK deploy, populate `apps/tarot/.env.local` from CloudFormation outputs (see `docs/cognito_expo_config_contract.md`)
- Auth logic lives in `apps/tarot/src/auth/` — `cognito-config.ts`, `auth-session.ts`, `use-auth.ts`, `token-storage.ts`

### UI Component Library

`packages/ui` is organized by atomic design: `stories/atoms/`, `molecules/`, `organisms/`, `screens/`, `templates/`

- Every component has a paired `.stories.tsx` file; MDX docs are optional
- MSW (`msw`) is set up for mocking GraphQL in Storybook
- Built and deployed to GitHub Pages via CI (`yarn build-storybook`)

## Conventions

### Commits

This repo uses **Conventional Commits** and **semantic-release** for automated versioning. Format:

```
type(scope): short description
```

Types that trigger releases: `fix` (patch), `feat` (minor), `feat!`/`fix!` (major).
Types that do not: `docs`, `test`, `chore`, `refactor`, `build`, `ci`.
Scope should be the workspace name (e.g., `ui`, `tarot`, `graph-api`, `hooks`, `infra`).

### TypeScript

- Shared type resolution is via workspace `*` references; `packages/hooks` and `packages/ui` are `"type": "module"`
- `packages/ui` sources directly from `index.tsx` (no pre-build step needed for the tarot app)
- `packages/hooks` requires a `build-types` step to produce `dist/` before tests or tarot app imports resolve

### Expo

**Read the versioned Expo docs at `https://docs.expo.dev/versions/v56.0.0/` before writing any Expo code** — APIs change significantly between SDK versions and this repo is on SDK 56.

### Knip

`yarn knip` detects unused exports. Files tagged `@lintignore` are excluded. Run it before PRs to avoid dead code.
