# AvatarImage Component

Displays a randomly selected tarot-themed avatar image fetched from the REST API.

## Overview

- Accepts an `apiBaseUrl` prop and calls `GET ${apiBaseUrl}/avatars` via `useAvatarImage` to load a pool of images.
- Renders the selected image through `AvatarDisplay`, keeping the visual atom shared with RSC-backed avatars.
- Press on the image to cycle to a new random image from the loaded pool.
- Long press is stubbed for a future save-to-profile feature.

## Props

- `apiBaseUrl`: Base URL for the REST API (e.g. `http://localhost:4100`). Required. Pass an empty string to skip fetching and display the default grey card.
- `size`: Size of the avatar image, default is `xlarge`.
- `saved`: Optional URL. If provided, the component bypasses the API and displays this image directly.

## State

- `hasSaved`: Initializes from the `saved` prop; pressing the avatar clears it so cycling resumes.
- `displayImage`: The URL currently rendered. Changes on each press without a new API call — cycling draws from the pool already held in hook state.

## Hook: useAvatarImage

```typescript
import { useAvatarImage } from '@simpletarot/hooks/client';

const { avatarImage, error, getAvatarImage, getNewAvatarImage, saveAvatarImage } =
    useAvatarImage(apiBaseUrl);
```

The hook calls `GET ${apiBaseUrl}/avatars` once on mount (skipped when `apiBaseUrl` is empty), stores the full `thumbnails` array in local state, and picks a random entry as the initial `avatarImage`. `getNewAvatarImage` cycles within that same array without a second network request.

### Returns

- `avatarImage`: Current image URL (defaults to the grey card placeholder until the fetch resolves).
- `error`: `Error | undefined` — set if the fetch fails or returns a non-2xx status.
- `getAvatarImage`: Returns the current `avatarImage` from state.
- `getNewAvatarImage`: Picks a new random URL from the loaded pool and updates `avatarImage`.
- `saveAvatarImage`: Stubbed — logs on long press; save-to-profile not yet implemented.
