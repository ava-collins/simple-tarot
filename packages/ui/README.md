## Overview

This package provides the Storybook-driven React Native component library shared by the mobile app â€” screens, organisms, molecules, and atoms for account auth and tarot reading flows.

## Contents

`stories/atoms` provides the base building blocks: `Card`, `CardDeck`, `CardDeal`, `Background`, `StartArrow`, `AvatarImage`, `FormButton`, `FormInput`, and `FormErrorText`.

`stories/molecules` provides `FormInputRow`, `QuickNav`, and `ShuffleCard`, composed from atoms.

`stories/organisms` provides `LoginForm`, `SignupForm`, `ForgotPasswordForm`, `UserAccount`, `Shuffle`, `DealCards`, and `NewReading`, composed from atoms and molecules.

`stories/screens` provides the top-level screens consumed by `apps/tarot`: `AccountScreen`, `LoginScreen`, `SignupScreen`, `ForgotPasswordScreen`, `CognitoSignInScreen`, `AuthCallbackScreen`, `SignOutScreen`, `LogoutCallbackScreen`, and `HomeScreen`.

`stories/templates` provides `MobileView`, the shared screen layout wrapper.

## Components

Components are organized by atomic design level (atoms â†’ molecules â†’ organisms â†’ screens â†’ templates) and documented in Storybook alongside their stories (`*.stories.tsx`) and usage docs (`*.mdx`).

`index.tsx` re-exports the screens and shared atoms consumed by `apps/tarot`. Molecules, organisms, and most atoms are internal building blocks composed by screens rather than exported directly â€” import components from `@simpletarot/ui` rather than reaching into `stories/`.

Screens compose hooks from `@simpletarot/hooks` and card components from `@simpletarot/cards` with the atoms/molecules/organisms in this package.

## Validation

This package has no test suite and is excluded from `yarn test`. After changing a component, validate through Storybook and type checking:

```sh
yarn workspace @simpletarot/ui storybook
yarn workspace @simpletarot/ui build-types
yarn lint
```
