# Simple Tarot Mobile App

`apps/tarot` is the Expo React Native mobile app for Simple Tarot. It uses Expo
Router and consumes shared code from the workspace packages.

## Expo Version

This app targets Expo SDK 56. Before changing Expo-specific APIs or config,
read the SDK 56 docs linked from `apps/tarot/AGENTS.md`.

## Cognito Public Config

The mobile app receives Cognito configuration through Expo public environment
variables. These values are public identifiers, URLs, and route paths; they are
not secrets. Do not add AWS credentials, Cognito client secrets, or private API
keys to `EXPO_PUBLIC_*` variables.

Create the local env file from the example:

```sh
cp apps/tarot/.env.local.example apps/tarot/.env.local
```

Then copy values from the deployed CDK stack outputs into `apps/tarot/.env.local`.
The full mapping is documented in
`docs/cognito_expo_config_contract.md`.

For preview and testing builds, create the same `EXPO_PUBLIC_*` values in the
appropriate EAS environment.

## Commands

Run commands from the repository root:

```sh
yarn workspace tarot start
yarn workspace tarot ios
yarn workspace tarot android
yarn workspace tarot web
yarn workspace tarot lint
```

## Notes

- The app scheme is `tarot`.
- Cognito callback/logout URLs must match the redirect URIs generated by the
  Expo auth runtime exactly.
- Auth screens and auth runtime code should stay in `apps/tarot` until a second
  consumer exists.
