# Reading API

`apps/graph-api` is the backend REST API for Simple Tarot. It generates AI-powered tarot readings using AWS Bedrock Knowledge Base (RAG) with Claude models.

## Endpoint

`POST /reading`

**Request:**
```json
{
  "items": [
    { "cardIndex": 0, "spreadPositionIndex": 0, "reversed": false },
    { "cardIndex": 7, "spreadPositionIndex": 1, "reversed": true }
  ]
}
```

**Response:**
```json
{
  "cards": [
    {
      "positionDisplayName": "Situation",
      "positionDescription": "The central theme of the reading...",
      "spreadPositionIndex": 0,
      "card": { "index": 0, "name": "fool", "title": "The Fool", "reversed": false, "..." },
      "cardReading": "2-3 sentence AI-generated interpretation..."
    }
  ],
  "synthesis": "3-4 sentence holistic narrative across all cards..."
}
```

`cardIndex` is 0–77. `spreadPositionIndex` is 0–9 (Celtic Cross). Both return 400 with a specific error message if out of range.

## Architecture

- Express v5 REST server (`src/index.ts`)
- `POST /reading` handler in `src/routes/reading.ts`
- `BedrockService` in `src/services/bedrock.ts` — KB retrieval (Haiku) + synthesis (Sonnet)
- Card metadata loaded from `src/data/cards.json` (generated by `yarn corpus:build`)
- Spread position metadata loaded from `src/data/spread-positions.json` (generated by `yarn corpus:build`)
- Corpus documents in `corpus/` (78 card docs + 10 position docs) — ingested into Bedrock KB via `yarn corpus:upload`

## Environment Variables

Populate `apps/graph-api/.env` from CDK stack outputs after deploying `SimpleTarotBedrockKb-<env>`:

```sh
NODE_ENV=development
DEV_PORT=4000
PROD_PORT=443

AWS_REGION=us-east-1
CORPUS_BUCKET_NAME=          # CDK output: CorpusBucketName
BEDROCK_KB_ID=               # CDK output: BedrockKnowledgeBaseId
BEDROCK_KB_DATA_SOURCE_ID=   # CDK output: BedrockDataSourceId

BEDROCK_MODEL_ID_HAIKU=us.anthropic.claude-haiku-4-5-20251001
BEDROCK_MODEL_ID_SONNET=us.anthropic.claude-sonnet-4-6
```

AWS credentials come from the environment (`AWS_PROFILE`, `AWS_ACCESS_KEY_ID`/`AWS_SECRET_ACCESS_KEY`, or an EC2/ECS instance role).

## How to Run

```sh
# 1. Build the corpus (first time or when source data changes)
yarn corpus:build

# 2. Upload corpus to S3 and sync Bedrock Knowledge Base
yarn corpus:upload

# 3. Start the dev server
yarn graph:dev
```

The dev server runs at `http://localhost:4000`.

## Corpus Management

Corpus source: `assets/ignore/corpus-source.json` (Firebase/Firestore export, read-only).

```sh
yarn corpus:build   # Regenerate corpus/ and src/data/ from source
yarn corpus:upload  # Upload to S3 + trigger Bedrock KB ingestion sync (polls to completion)
```

The corpus must be rebuilt and re-uploaded any time `corpus-source.json` changes.
